# Security Policy

## Security Baseline
- TLS 1.2+ for all internal and external communications
- AES-256-class encryption for sensitive records at rest
- Signed webhook validation
- Least-privilege access controls
- No raw biometric image storage in normal operational systems

## Sensitive Components
- biometric proof verification
- identity binding records
- consent artifacts
- transaction orchestration
- settlement callbacks
- partner credentials and secrets
- audit logs

## Reporting
Do not open public issues for sensitive vulnerabilities. Route reports privately to the project security contact or project administrator.
